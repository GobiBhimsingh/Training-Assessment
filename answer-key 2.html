Section 1 XPath Fundamentals Q1 – Q5
Q1 Relative XPath — Input field Write XPath
HTML Snippet
<form id="loginForm">
  <label for="username">Username</label>
  <input type="text" id="username" name="userName">
  <label for="password">Password</label>
  <input type="password" id="password" name="password">
  <button type="submit">Login</button>
</form>

Write an XPath to select the username input field.

✔ Accepted Answers
  • 1.//input[@id='username']Primary
  • 2.//input[@name='userName']
  • 3.//form[@id='loginForm']//input[@type='text']
  • 4.//input[@type='text'][@id='username']
Q2 Absolute XPath Multiple Choice
HTML Snippet
<html><body><div><form><input type="text" id="email"></form></div></body></html>

Which is a valid absolute XPath to reach the email input?

✔ Correct Answer
A /html/body/div/form/input
Q3 Select by Attribute Write XPath
HTML Snippet
<div class="actions">
  <button type="submit">Login</button>
  <button type="button">Cancel</button>
</div>

Write an XPath to select only the submit button using its type attribute.

✔ Accepted Answers
  • 1.//button[@type='submit']Primary
  • 2.//div[@class='actions']/button[@type='submit']
  • 3.//div/button[@type='submit']
Q4 Select by text() Multiple Choice
HTML Snippet
<div class="actions">
  <button type="submit">Login</button>
  <button type="button">Cancel</button>
</div>

Pick the correct XPath to click the Login button by its visible text.

✔ Correct Answer
B //button[text()='Login']
Q5 contains() — Class attribute Write XPath
HTML Snippet
<div class="nav-menu primary-nav">
  <a href="/home">Home</a>
  <a href="/about">About</a>
  <a href="/contact">Contact</a>
</div>

The div class always contains nav-menu. Write an XPath to select this div.

✔ Accepted Answers
  • 1.//div[contains(@class,'nav-menu')]Primary
  • 2.//div[contains(@class, 'nav-menu')]
  • 3.//div[contains(@class,'primary-nav')]
  • 4.//div[contains(@class, 'primary-nav')]
Section 2 Predicates, Conditions & Functions Q6 – Q11
Q6 starts-with() — Dynamic IDs Multiple Choice
HTML Snippet
<table>
  <tr>
    <td id="row_001">Alice</td>
    <td id="row_002">Bob</td>
    <td id="row_003">Charlie</td>
  </tr>
</table>

IDs always start with row_. Which XPath selects all such cells?

✔ Correct Answer
C //td[starts-with(@id,'row_')]
Q7 AND condition Write XPath
HTML Snippet
<form id="signupForm">
  <input type="text" name="firstName" placeholder="First Name">
  <input type="text" name="lastName"  placeholder="Last Name">
  <input type="email" name="email"    placeholder="Email">
  <input type="submit" value="Register">
</form>

Write an XPath to select the input with both type='text' and name='firstName'.

✔ Accepted Answers
  • 1.//input[@type='text' and @name='firstName']Primary
  • 2.//input[@name='firstName' and @type='text']
  • 3.//form//input[@type='text' and @name='firstName']
Q8 OR condition Write XPath
HTML Snippet
<form id="registerForm">
  <input type="text"     id="username" placeholder="Username">
  <input type="password" id="password" placeholder="Password">
  <input type="email"    id="email"    placeholder="Email">
</form>

Write an XPath to select inputs whose type is either text or email.

✔ Accepted Answers
  • 1.//input[@type='text' or @type='email']Primary
  • 2.//form//input[@type='text' or @type='email']
  • 3.//form[@id='registerForm']//input[@type='text' or @type='email']
Q9 Indexing Write XPath
HTML Snippet
<ul id="menuList">
  <li>Dashboard</li>
  <li>Reports</li>
  <li>Settings</li>
  <li>Logout</li>
</ul>

Write an XPath to select the third list item (Settings) using index.

✔ Accepted Answers
  • 1.//ul[@id='menuList']/li[3]Primary
  • 2.(//li)[3]
  • 3.//ul/li[3]
Q10 last() function Multiple Choice
HTML Snippet
<ul id="steps">
  <li>Step 1: Fill form</li>
  <li>Step 2: Review</li>
  <li>Step 3: Submit</li>
</ul>

Which XPath selects the last list item regardless of count?

✔ Correct Answer
C //ul/li[last()]
Q11 position() function Write XPath
HTML Snippet
<table id="dataTable">
  <tr><td>Row 1</td></tr>
  <tr><td>Row 2</td></tr>
  <tr><td>Row 3</td></tr>
</table>

Write an XPath to select the second <tr> using position().

✔ Accepted Answers
  • 1.//table[@id='dataTable']/tr[position()=2]Primary
  • 2.//tr[position()=2]
  • 3.//table/tr[position()=2]

Q12 parent:: axis Multiple Choice
HTML Snippet
<div class="form-group">
  <label for="city">City</label>
  <input type="text" id="city" name="city">
</div>

You have located input[@id='city']. Which XPath navigates to its parent div?

✔ Correct Answer

Q14 ancestor:: axis Write XPath
HTML Snippet
<div id="wrapper">

    <form id="contactForm">
      <input type="text" id="fullName">
    </form>
  </section>

Starting from input[@id='fullName'], use ancestor:: to reach div#wrapper.

✔ Accepted Answers

  • 2.//input[@id='fullName']/ancestor::div

Which XPath uses descendant:: to select all links inside div#sidebar?

Q16 following-sibling:: axis Write XPath
HTML Snippet
<div class="field-row">
  <label for="dob">Date of Birth</label>
  <input type="date" id="dob" name="dob">
</div>

You know the label for='dob'. Use following-sibling:: to reach the input next to it.

✔ Accepted Answers
  • 1.//label[@for='dob']/following-sibling::inputPrimary
  • 2.//label[@for='dob']/following-sibling::input[@id='dob']
Q17 preceding-sibling:: axis Multiple Choice
HTML Snippet
<tr>
  <td>Product Name</td>
  <td>Quantity</td>
  <td class="price">$49.99</td>
</tr>

You are on td.price. Which XPath selects the cell immediately before it (Quantity)?

✔ Correct Answer
B //td[@class='price']/preceding-sibling::td[1]
Q18 following:: axis Write XPath
HTML Snippet
<div id="page">
  <h2 id="sectionA">Section A</h2>
  <p>Some intro text.</p>
  <h2 id="sectionB">Section B</h2>
  <p>More content here.</p>
</div>

Write an XPath using following:: to select all elements after h2#sectionA.

✔ Accepted Answers
  • 1.//h2[@id='sectionA']/following::*Primary
  • 2.//h2[@id='sectionA']/following::node()
Q19 preceding:: axis Write XPath
HTML Snippet
<form id="checkoutForm">
  <input type="text"   id="cardName"   placeholder="Name on Card">
  <input type="text"   id="cardNumber" placeholder="Card Number">
  <input type="submit" id="payBtn"     value="Pay Now">
</form>

Write an XPath using preceding:: to select all elements before the submit input id='payBtn'.

✔ Accepted Answers
  • 1.//input[@id='payBtn']/preceding::*Primary
  • 2.//input[@value='Pay Now']/preceding::*
  • 3.//input[@id='payBtn']/preceding::input
Q20 self:: axis Write XPath

<button id="saveBtn" class="btn-primary" type="button">Save</button>

Write an XPath using self:: starting from //button[@id='saveBtn'].

✔ Accepted Answers

Section 4 Real-World Elements & Scenarios Q21 – Q25
Q21 Checkboxes Multiple Choice
HTML Snippet
<div class="preferences">
  <input type="checkbox" id="chk_news"   name="subscribe" value="news">
  <label for="chk_news">Newsletter</label>
  <input type="checkbox" id="chk_offers" name="subscribe" value="offers">
  <label for="chk_offers">Special Offers</label>
</div>

Which XPath selects all checkbox inputs inside the preferences div?

✔ Correct Answer
B //div[@class='preferences']//input[@type='checkbox']
Q22 Tables — Row selection Write XPath
HTML Snippet
<table id="userTable">
  <thead><tr><th>Name</th><th>Role</th><th>Status</th></tr></thead>
  <tbody>
    <tr><td>Alice</td><td>Admin</td>  <td>Active</td></tr>
    <tr><td>Bob</td>  <td>User</td>   <td>Inactive</td></tr>
    <tr><td>Carol</td><td>User</td>   <td>Active</td></tr>
  </tbody>
</table>

Write an XPath to select the entire row that contains the name Bob.

✔ Accepted Answers
  • 1.//tr[td[text()='Bob']]Primary
  • 2.//table[@id='userTable']//tr[td[text()='Bob']]
  • 3.//tbody/tr[td[text()='Bob']]
Q23 Links — contains() on text Multiple Choice
HTML Snippet
<footer>
  <a href="/privacy-policy">Privacy Policy</a>
  <a href="/terms-of-service">Terms of Service</a>
  <a href="/contact-us">Contact Us</a>
</footer>

Which XPath selects the link whose text contains the word Privacy?

✔ Correct Answer
C //a[contains(text(),'Privacy')]
Q24 Labels and Inputs Write XPath
HTML Snippet
<div class="form-group">
  <label for="phoneNumber">Phone</label>
  <input type="tel" id="phoneNumber" name="phone" placeholder="+1 000 000 0000">
</div>

Find the input by locating its label text Phone and navigating to the input via following-sibling::.

✔ Accepted Answers
  • 1.//label[text()='Phone']/following-sibling::inputPrimary
  • 2.//label[contains(text(),'Phone')]/following-sibling::input
  • 3.//input[@id='phoneNumber']
Q25 Dynamic IDs — Nested spans Multiple Choice
HTML Snippet
<div class="product-card">
  <span class="product-title">
    <span class="badge new-badge">NEW</span>
    Wireless Headphones
  </span>
  <span class="product-price">$89.99</span>
  <button id="addToCart_8472" class="add-btn">Add to Cart</button>
</div>

The button ID is dynamic. Which XPath reliably targets it regardless of the number?

✔ Correct Answer
B //button[contains(@id,'addToCart_')]
