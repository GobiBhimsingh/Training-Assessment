Section 1 Advanced XPath Functions Q1 – Q6
Q1 normalize-space() Function Write XPath
HTML Snippet
<div class="checkout-panel">
  <button id="placeOrder">
    Place   Order
  </button>
  <button id="cancelOrder">Cancel</button>
</div>

Write an XPath using normalize-space() to reliably locate the Place Order button.

✔ Accepted Answers
  • 1. //button[normalize-space()='Place Order'] Primary
  • 2. //button[normalize-space(text())='Place Order']
  • 3. //button[normalize-space(.)='Place Order']
  • 4. //div[@class='checkout-panel']/button[normalize-space()='Place Order']
💡 Concept normalize-space() strips leading/trailing whitespace and collapses all internal whitespace sequences into a single space. It is essential when button/label text contains newlines or multiple spaces from template rendering.
Q2 not() Predicate Multiple Choice
HTML Snippet
<form id="profileForm">
  <input type="text"  id="fname"  placeholder="First Name">
  <input type="text"  id="lname"  placeholder="Last Name"  disabled>
  <input type="email" id="email"  placeholder="Email">
  <input type="tel"   id="phone"  placeholder="Phone"      disabled>
</form>

Which XPath selects only the input fields that are not disabled?

✔ Correct Answer
B //input[not(@disabled)]
💡 Concept not(@disabled) checks that the disabled attribute is absent. The presence of the attribute alone (even with no value or value="disabled") means the element is disabled. Using @disabled=false is incorrect because XPath compares it as a string and the attribute value is not the string "false".
Q3 XPath Union Operator ( | ) Write XPath
HTML Snippet
<div class="modal-footer">
  <button type="submit"  class="btn-primary">Confirm</button>
  <button type="reset"   class="btn-secondary">Reset</button>
  <a href="/help"        class="btn-link">Help</a>
</div>

Write a single XPath using | to select both the Confirm button and the Help link.

✔ Accepted Answers
  • 1. //button[@type='submit'] | //a[@href='/help'] Primary
  • 2. //button[text()='Confirm'] | //a[text()='Help']
  • 3. //button[@class='btn-primary'] | //a[@class='btn-link']
  • 4. //button[normalize-space()='Confirm'] | //a[normalize-space()='Help']
💡 Concept The | union operator merges two node-sets into one. It is the only set union operator in XPath 1.0. It is useful to reduce driver calls — instead of two findElements calls you get both in one. Order of results follows document order.
Q4 translate() — Case-Insensitive Search Multiple Choice
HTML Snippet
<nav class="main-menu">
  <a href="/dashboard">DASHBOARD</a>
  <a href="/reports">Reports</a>
  <a href="/settings">settings</a>
</nav>

Which XPath uses translate() to find the Dashboard link regardless of text case?

✔ Correct Answer
C //a[translate(text(),'ABCDEFGHIJKLMNOPQRSTUVWXYZ','abcdefghijklmnopqrstuvwxyz')='dashboard']
💡 Concept translate(string, from-chars, to-chars) replaces characters one-for-one. Using it to convert text to lowercase allows case-insensitive matching. Note: lower-case() is XPath 2.0 and is not universally supported in Selenium drivers.
Q5 count() Function Write XPath
HTML Snippet
<table id="ordersTable">
  <thead><tr><th>Order ID</th>...</tr></thead>
  <tbody>
    <tr>...</tr>
    <tr>...</tr>
    <tr>...</tr>
  </tbody>
</table>

Write an XPath using count() to return the total number of data rows (tbody only).

✔ Accepted Answers
  • 1. count(//table[@id='ordersTable']/tbody/tr) Primary
  • 2. count(//table[@id='ordersTable']//tbody/tr)
  • 3. count(//tbody/tr)
  • 4. count(//table/tbody/tr)
💡 Concept count(node-set) returns the number of nodes. Using tbody/tr precisely targets data rows only. In Selenium: driver.findElements(By.xpath("//tbody/tr")).size() is the equivalent programmatic check.
Q6 string-length() Function Multiple Choice
HTML Snippet
<ul id="productList">
  <li data-sku="AB">Short SKU</li>
  <li data-sku="XYZABC123">Long SKU</li>
  <li data-sku="MN">Short SKU</li>
</ul>

Which XPath selects items whose data-sku has a length > 4 characters?

✔ Correct Answer
C //li[string-length(@data-sku) > 4]
💡 Concept string-length(string) returns the number of characters. Works on attribute values, text nodes, or concatenated strings. Useful for validation assertions or filtering elements with long/short identifiers.
Section 2 Modern HTML Attributes (data-* & ARIA) Q7 – Q13
Q7 data-testid — Test Automation Attribute Write XPath
HTML Snippet
<div class="login-container">
  <input data-testid="email-input"    type="email">
  <input data-testid="password-input" type="password">
  <button data-testid="login-submit"  type="submit">Sign In</button>
</div>

Write an XPath to locate the Sign In button using its data-testid value.

✔ Accepted Answers
  • 1.//button[@data-testid='login-submit']Primary
  • 2.//*[@data-testid='login-submit']
  • 3.//button[@data-testid='login-submit'][@type='submit']
💡 Concept data-testid is added by developers exclusively for test automation. It is the most stable locator — it does not change with styling, layout, text content, or ID regeneration. This is the recommended primary locator strategy in modern teams.
Q8 data-cy — Cypress-Style Test Attribute Multiple Choice
HTML Snippet
<form data-cy="signup-form">
  <input data-cy="name-field"  type="text">
  <input data-cy="email-field" type="email">
  <select data-cy="country-select">...</select>
  <button data-cy="signup-submit">Create Account</button>
</form>

Which XPath locates the country dropdown using data-cy?

✔ Correct Answer
B //select[@data-cy='country-select']
💡 Concept Custom data-* attributes use the @ prefix like standard attributes. Common test attribute names: data-testid (React Testing Library), data-cy (Cypress), data-qa, data-test. All are accessed with [@data-xxx='value'].
Q9 ARIA — aria-label Attribute Write XPath
HTML Snippet
<header class="app-header">
  <button aria-label="Open navigation menu">☰</button>
  <button aria-label="Search"><svg/></button>
  <button aria-label="Close dialog">✕</button>
</header>

Write an XPath to locate the Close dialog button using aria-label.

✔ Accepted Answers
  • 1.//button[@aria-label='Close dialog']Primary
  • 2.//header//button[@aria-label='Close dialog']
  • 3.//*[@aria-label='Close dialog']
💡 Concept aria-label provides an accessible name for elements with no visible text (icon buttons, close buttons, hamburger menus). It is a reliable locator for icon-only controls and is preferred over fragile XPath based on icon characters or SVG content.
Q10 ARIA — aria-placeholder Attribute Multiple Choice
HTML Snippet
<div role="textbox"
     aria-placeholder="Search for products..."
     aria-label="Product search"
     contenteditable="true"
     id="searchBox">
</div>

Which XPath locates this custom search box using aria-placeholder?

✔ Correct Answer
B //div[@aria-placeholder='Search for products...']
💡 Concept Custom components built with div/span use ARIA attributes instead of native HTML attributes. aria-placeholder is the ARIA equivalent of the placeholder attribute. Always check if a component is native HTML or custom-built before choosing your locator.
Q11 ARIA — aria-expanded (Accordion) Write XPath
HTML Snippet
<div class="accordion">
  <button aria-expanded="true"  aria-controls="panel1">Billing Details</button>
  <button aria-expanded="false" aria-controls="panel2">Shipping Address</button>
  <button aria-expanded="false" aria-controls="panel3">Payment Method</button>
</div>

Write an XPath to select the accordion header that is currently expanded.

✔ Accepted Answers
  • 1.//button[@aria-expanded='true']Primary
  • 2.//button[@class='acc-header'][@aria-expanded='true']
  • 3.//button[contains(@class,'acc-header') and @aria-expanded='true']
💡 Concept aria-expanded='true' indicates the controlled section is visible. Use this to assert accordion/dropdown state in tests. Never rely on CSS classes like "active" or "open" — they may differ across frameworks.
Q12 ARIA — aria-selected (Tab Panels) Multiple Choice
HTML Snippet
<div role="tablist">
  <button role="tab" aria-selected="true"  id="tab-summary">Summary</button>
  <button role="tab" aria-selected="false" id="tab-detail">Detail</button>
  <button role="tab" aria-selected="false" id="tab-export">Export</button>
</div>

Which XPath selects the currently active / selected tab?

✔ Correct Answer
B //button[@aria-selected='true']
💡 Concept aria-selected='true' marks the active tab. After clicking a tab in a test, assert getAttribute("aria-selected") equals "true" to verify the tab switch was successful.
Q13 ARIA — aria-checked (Custom Checkbox) Write XPath
HTML Snippet
<div class="settings-panel">
  <div role="checkbox" aria-checked="true"  data-setting="notifications">Email Notifications</div>
  <div role="checkbox" aria-checked="false" data-setting="sms">SMS Alerts</div>
  <div role="checkbox" aria-checked="true"  data-setting="newsletter">Newsletter</div>
</div>

Write an XPath to select all custom checkbox divs that are currently checked.

✔ Accepted Answers
  • 1.//*[@role='checkbox'][@aria-checked='true']Primary
  • 2.//div[@role='checkbox'][@aria-checked='true']
  • 3.//div[@aria-checked='true']
  • 4.//*[@aria-checked='true']
💡 Concept Custom checkboxes use role="checkbox" and aria-checked. Never use @checked (only works on native <input type="checkbox">). Asserting aria-checked='true' is the correct way to verify custom toggle state.
Section 3 Special Input States Q14 – Q17
Q14 Hidden Input Fields Multiple Choice
HTML Snippet
<form id="paymentForm">
  <input type="hidden" name="csrf_token"   value="x9aK3mZ...">
  <input type="hidden" name="session_id"   value="sess_abc123">
  <input type="hidden" name="amount_cents" value="4999">
  <button type="submit">Pay $49.99</button>
</form>

Which XPath locates the hidden input storing the CSRF token?

✔ Correct Answer
A //input[@type='hidden' and @name='csrf_token']
💡 Concept Hidden inputs (type='hidden') are in the DOM and fully locatable. They carry security tokens, session IDs, and form metadata. Reading hidden field values is critical in security and integration tests.
Q15 Disabled Elements Write XPath
HTML Snippet
<div class="wizard-nav">
  <button id="btnBack"   type="button" disabled>← Back</button>
  <button id="btnNext"   type="button">Next →</button>
  <button id="btnSubmit" type="submit" disabled>Submit</button>
</div>

Write an XPath to select all disabled buttons in this wizard navigation bar.

✔ Accepted Answers
  • 1.//button[@disabled]Primary
  • 2.//div[@class='wizard-nav']/button[@disabled]
  • 3.//button[@disabled='disabled']
  • 4.//div//button[@disabled]
💡 Concept @disabled checks attribute presence (boolean attribute). You can assert a button is disabled in a test by verifying getAttribute("disabled") != null or using isEnabled() == false in Selenium.
Q16 Read-Only Input Fields Multiple Choice
HTML Snippet
<form class="user-profile">
  <input type="text"  id="username"   value="john_doe"  readonly>
  <input type="email" id="email"      value="john@..."  readonly>
  <input type="text"  id="department" value="Engineering">
  <input type="tel"   id="phone"      value="+1-555-0100">
</form>

Which XPath selects only the read-only input fields?

✔ Correct Answer
C //input[@readonly]
💡 Concept readonly is a boolean HTML attribute — check its presence with @readonly. Read-only fields are visible, selectable, and copyable but cannot be edited by the user. In Selenium, isEnabled() returns true for readonly fields (unlike disabled), so attribute checking is essential.
Q17 File Upload Input Write XPath
HTML Snippet
<div class="upload-section">
  <input type="file" id="avatarUpload"  name="avatar"  accept="image/png,image/jpeg" data-testid="avatar-file-input">
  <input type="file" id="resumeUpload"  name="resume"  accept=".pdf,.doc"            data-testid="resume-file-input">
</div>

Write an XPath to locate the resume file upload input using its name attribute.

✔ Accepted Answers
  • 1.//input[@type='file'][@name='resume']Primary
  • 2.//input[@name='resume']
  • 3.//input[@type='file' and @name='resume']
  • 4.//input[@data-testid='resume-file-input']
💡 Concept File inputs are located like any other input. In Selenium, use sendKeys("/absolute/path/to/file.pdf") to set the file — do NOT click the OS file picker dialog. The path must be absolute on the machine running the test.
Section 4 Complex DOM Structures Q18 – Q21
Q18 Select Dropdown — Option by Value Multiple Choice
HTML Snippet
<select id="roleSelect" name="role">
  <option value="">-- Select Role --</option>
  <option value="viewer">Viewer</option>
  <option value="editor">Editor</option>
  <option value="admin">Administrator</option>
  <option value="owner">Owner</option>
</select>

Which XPath locates the Administrator option?

✔ Correct Answer
D All of the above are valid
💡 Concept Options are locatable by text or value. In Selenium, prefer the Select class: new Select(el).selectByValue("admin") or .selectByVisibleText("Administrator"). Directly clicking an option element via XPath also works but the Select class is cleaner.
Q19 Table — Select Cell by Column Header Name Write XPath
HTML Snippet
<table id="inventoryTable">
  <thead><tr><th>Item</th><th>Category</th><th>Stock</th><th>Price</th></tr></thead>
  <tbody>
    <tr><td>Wireless Mouse</td><td>Electronics</td><td>142</td><td>$29.99</td></tr>
    <tr><td>USB-C Hub</td>    <td>Electronics</td><td>57</td> <td>$49.99</td></tr>
  </tbody>
</table>

Write an XPath to select the Price cell from the row where the Item is USB-C Hub.

✔ Accepted Answers
  • 1.//tr[td[1][text()='USB-C Hub']]/td[count(//th[text()='Price']/preceding-sibling::th)+1]Dynamic
  • 2.//tbody/tr[td[text()='USB-C Hub']]/td[4]
  • 3.//tr[td[text()='USB-C Hub']]/td[last()]
💡 Concept The dynamic approach uses count(//th[text()='Price']/preceding-sibling::th)+1 to calculate column position from the header name — robust even if columns are reordered. The hardcoded td[4] is fragile. Always prefer dynamic column index in production test suites.
Q20 contenteditable — Rich Text Editor Multiple Choice
HTML Snippet
<div id="editorBody"
     role="textbox"
     contenteditable="true"
     aria-label="Message body"
     class="editor-area">
</div>

Which is the most reliable XPath for this rich text editor?

✔ Correct Answer
D All of the above are valid and reliable
💡 Concept All three work. Locator priority: data-testid > aria-label > id > contenteditable. Using contenteditable='true' alone is risky if multiple editors exist. In Selenium, type into a contenteditable div using sendKeys() or JavaScript element.innerHTML = text.
Q21 SVG Elements — local-name() Write XPath
HTML Snippet
<div class="chart-container">
  <svg id="barChart" width="400" height="300" role="img" aria-label="Sales Chart">
    <g class="bars">
      <rect class="bar" x="10" y="50" width="40" height="150"/>
    </g>
  </svg>
</div>

Write an XPath using local-name() to locate the SVG element by id barChart.

✔ Accepted Answers
  • 1.//*[local-name()='svg'][@id='barChart']Primary
  • 2.//*[@id='barChart']
  • 3.//svg[@id='barChart']
💡 Concept SVG elements are in the SVG namespace. local-name() ignores namespaces and matches only the local tag name. //*[@id='barChart'] also works because it uses a wildcard. For inner SVG elements like <rect> or <path>, always use //*[local-name()='rect'].
Section 5 Framework Patterns & Advanced Combinations Q22 – Q25
Q22 React / Framework — Custom data-* Attributes Multiple Choice
HTML Snippet
<ul data-component="TodoList">
  <li data-index="0" data-completed="false" data-id="todo_1">Buy groceries</li>
  <li data-index="1" data-completed="true"  data-id="todo_2">Send report</li>
  <li data-index="2" data-completed="false" data-id="todo_3">Book flight</li>
</ul>

Which XPath selects to-do items that are not yet completed?

✔ Correct Answer
A //li[@data-completed='false']
💡 Concept data-* attribute values are always DOM strings. Use @data-completed='false' (string) not @data-completed=false (boolean, incorrect in XPath 1.0). This is a common bug when writing XPath for React/Vue/Angular rendered components.
Q23 Wildcard * and @* (Any Element / Any Attribute) Write XPath
HTML Snippet
<section class="user-card">
  <div class="card-body">
    <span data-field="email">alice@example.com</span>
    <span data-field="role">Administrator</span>
    <span data-field="joined">2022-03-15</span>
  </div>
</section>

Write an XPath using * wildcard to select any element with data-field='role'.

✔ Accepted Answers
  • 1.//*[@data-field='role']Primary
  • 2.//section//*[@data-field='role']
  • 3.//div[@class='card-body']//*[@data-field='role']
💡 Concept * matches any element tag name. //*[@data-field='role'] selects the span without needing to know it is a span. Use when the tag name may change between releases or implementations. @* can also match any attribute: //*[@*='role'] selects any element having any attribute with value 'role'.
Q24 not( contains() ) — Exclude by Partial Class Multiple Choice
HTML Snippet
<nav class="breadcrumb">
  <a href="/home"     class="crumb-link">Home</a>
  <a href="/products" class="crumb-link">Products</a>
  <a href="/laptops"  class="crumb-link">Laptops</a>
  <span               class="crumb-link crumb-active">Dell XPS 15</span>
</nav>

Which XPath selects breadcrumb items excluding the one with class crumb-active?

✔ Correct Answer
B //*[@class='crumb-link'][not(contains(@class,'crumb-active'))]
💡 Concept not(contains(@class,'crumb-active')) excludes elements whose class attribute contains that substring. Using exact match @class='crumb-active' would fail because the active element has two classes: 'crumb-link crumb-active'. Always use contains() for multi-class elements.
Q25 Complex Multi-Predicate — Axes + Functions Combined Write XPath
HTML Snippet
<div class="form-section">
  <div class="field-group has-error">
    <input type="text" id="cardNum" aria-invalid="true">
    <span class="error-msg">Card number must be 16 digits.</span>
  </div>
  <div class="field-group">
    <input type="text" id="expiry" aria-invalid="false">
  </div>
  <div class="field-group has-error">
    <input type="text" id="cvv" aria-invalid="true">
    <span class="error-msg">CVV must be 3 digits.</span>
  </div>
</div>

Write an XPath to find all error message spans inside field groups that contain an input with aria-invalid='true'.

✔ Accepted Answers
  • 1.//div[contains(@class,'field-group')][input[@aria-invalid='true']]/span[@class='error-msg']Primary
  • 2.//div[@class='field-group has-error']/span[@class='error-msg']
  • 3.//div[contains(@class,'has-error')]/span[contains(@class,'error-msg')]
  • 4.//input[@aria-invalid='true']/following-sibling::span[@class='error-msg']
💡 Concept Answer 1 is the most semantically precise: it uses a parent predicate [input[@aria-invalid='true']] to filter containers, then navigates to the error span. Answer 4 uses following-sibling:: which is equally elegant. Answer 2 uses a hardcoded class combination which is fragile. Answer 3 is more flexible. All four are valid in this HTML context.
