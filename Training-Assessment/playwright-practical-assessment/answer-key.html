MCQ Questions — Answers & Explanations (Q36–Q50)

Q36 MCQ Answer: B
Playwright auto-waits before click(). The assertion on #result can still time out if the action completes later than the default 5s timeout — increase timeout or use waitForLoadState.
Q37 MCQ Answer: B
Official Playwright order: getByRole (mirrors accessibility) → getByTestId (explicit test hooks) → getByText/getByLabel → CSS/XPath as last resort.
Q38 MCQ Answer: B
Browser contexts are fully isolated environments. They share the browser process but have separate cookies, storage, sessions — like different incognito windows.
Q39 MCQ Answer: C
page.route() is the interception API — it can fulfill, abort, or continue requests. page.on('request') is observation-only and cannot block or modify the request.
Q40 MCQ Answer: C
fullyParallel:true breaks the file-level barrier — individual tests within a file can run in parallel across workers, not just different files.
Q41 MCQ Answer: C
Soft assertions collect failures, continue execution, then throw them all together at assertAll(). The test only fails after assertAll() is called.
Q42 MCQ Answer: C
scope:'worker' means the fixture is created once per worker process and shared by all tests running in that worker, reducing expensive setup cost.
Q43 MCQ Answer: C
networkidle waits until there are no network connections for 500ms. Ideal for SPAs that fire async fetch/XHR requests after the initial DOM load.
Q44 MCQ Answer: B
test.step() creates labeled groups in the HTML report and Trace Viewer. It has no effect on retries or browser context — it is purely for reporting.
Q45 MCQ Answer: B
Test-level configuration overrides global config. retries:0 at test level means zero retries for that specific test regardless of the global setting.
Q46 MCQ Answer: B
getByRole('button', {name:'Close dialog'}) matches by ARIA role + accessible name (aria-label). This is the most resilient and semantically correct approach.
Q47 MCQ Answer: C
on-first-retry records a trace only when a test fails the first time and enters its first retry cycle. No retry = no trace file.
Q48 MCQ Answer: B
globalSetup runs once before the entire suite in a separate Node.js process. It cannot use page/browser fixtures; use it for DB seeding, token generation, etc.
Q49 MCQ Answer: B
Playwright enforces strict mode — locators must match exactly one element during action calls. Multiple matches throw a StrictModeViolation error.
Q50 MCQ Answer: B
expect(fn).toPass() retries the entire async function block (including the fetch call) until all inner assertions pass or timeout. Unlike expect.poll() it can wrap multi-step logic.
